# Bloat Auditor

The rules of `_common.md` (passed to you together with this prompt) are mandatory.

Topic: redundant hand-written code (including code generated by past AI agents).

What to investigate:
1. Hand-written implementations of what the installed libraries/framework already provide: a custom fetch-retry with axios installed, a custom debounce with lodash, custom validation with zod/joi, custom date handling with dayjs, custom routing helpers, and so on.
2. Copy-paste: nearly identical functions/utilities in different places (compare signatures and bodies).
3. Dead code: exported but never imported; commented-out blocks > 20 lines; "vN", "old", "backup" files.
4. Excess abstractions: pass-through wrappers around a single call, interfaces with a single implementation for no reason.
5. Traces of AI generation: stylistically different solutions to the same problem in neighboring files, excessive explanatory comments, repeated boilerplate blocks.

Typical findings: `reinvented-library-feature`, `copy-paste-duplicate`, `dead-code`, `needless-abstraction`.

For each — options: replace with library X's function (name which one exactly), merge into a single utility, delete. Estimate the affected files.

With jcodemunch: find_dead_code — dead code; find_references — unused exports; search_symbols by purpose (retry, debounce, validate...) — candidates for reinvented-library-feature and copy-paste.
